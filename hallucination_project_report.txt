Midterm Report: Evaluating and Reducing Hallucination in Vision-Language Models
Introduction
In this project, I revised my original topic from a multi-view image generation system to a more focused and feasible problem: hallucination in vision-language models (VLMs). The initial multi-view project required complex 3D understanding and was difficult to implement within the course timeline. Therefore, I shifted to studying hallucination, which is a well-defined and actively researched problem in computer vision.  Vision-language models are designed to understand images and generate text descriptions. However, these models often produce hallucinated content, meaning they describe objects or attributes that are not actually present in the image. This issue is critical because it affects the reliability of AI systems in real-world applications such as autonomous systems, assistive technology, and content generation.  The goal of this project is to evaluate how frequently hallucination occurs in image captioning tasks and explore simple methods to reduce it. By focusing on a clear and measurable problem, this project aims to provide insights into model behavior and potential improvements.
Literature Review
Hallucination in vision-language models has become an important topic in recent research. Early image captioning systems used convolutional neural networks (CNNs) combined with recurrent neural networks (RNNs), such as LSTMs, to generate descriptions. These models already exhibited hallucination issues due to limited contextual understanding.  Recent models, including transformer-based architectures such as BLIP and CLIP-based systems, have improved performance but still suffer from hallucination. Researchers have proposed different approaches to study this problem, including benchmark datasets for hallucination detection, evaluation metrics, and methods to reduce hallucinated outputs.  Some studies focus on identifying hallucination by comparing generated captions with ground truth annotations. Others introduce structured prompting or constraint-based decoding to reduce incorrect outputs. Despite progress, hallucination remains an open problem, especially in complex or ambiguous scenes.
Data and Framework
This project will use the MS COCO dataset, which is widely used for image captioning tasks. The dataset contains images with multiple human-annotated captions, making it suitable for evaluating hallucination.  The implementation will be based on PyTorch and HuggingFace Transformers. Pretrained vision-language models such as BLIP or similar architectures will be used to generate captions. Using pretrained models allows efficient experimentation without requiring large-scale training.  Data preprocessing will involve selecting a subset of images and their corresponding captions. The dataset will be split into evaluation sets for consistent comparison across models and methods.
System Design
The system consists of three main components: image input, caption generation, and hallucination evaluation.  First, an image is provided as input to a vision-language model. The model generates a textual description of the image. Different models or configurations will be tested to compare their performance.  Second, the generated captions are analyzed to determine whether they contain hallucinated content. This involves comparing the generated captions with ground truth annotations or using rule-based checks to identify objects that are not present.  Third, the system applies simple strategies to reduce hallucination. For example, structured prompts can guide the model to focus on visible objects, and filtering methods can remove uncertain outputs.  This design allows systematic comparison between baseline and improved approaches.
Implementation Plan
The implementation will follow several steps. First, the dataset will be prepared by selecting a subset of images from MS COCO. Ground truth captions will be stored for evaluation.  Second, pretrained vision-language models will be loaded using HuggingFace Transformers. These models will generate captions for each image.  Third, hallucination detection will be implemented. This can include keyword matching, comparison with ground truth, or simple heuristic rules.  Fourth, different prompting strategies will be tested. For example, prompts that emphasize accuracy or restrict speculation may reduce hallucination.  Finally, results will be evaluated using metrics such as hallucination rate and caption quality. The findings will be analyzed to determine which methods are most effective.
Challenges and Future Work
One challenge in this project is defining hallucination clearly. Some descriptions may be partially correct or ambiguous, making evaluation difficult. Human judgment may be required in some cases.  Another challenge is that pretrained models may behave differently depending on prompts and input conditions. Ensuring fair comparison requires careful experimental design.  In future work, more advanced methods could be explored, such as fine-tuning models or incorporating grounding mechanisms to improve alignment between image and text. Additionally, larger datasets and more complex evaluation metrics could provide deeper insights.  Overall, this project focuses on a practical and relevant problem in modern computer vision and aims to contribute to understanding and reducing hallucination in vision-language models.
